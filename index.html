Help & Guide
Everything you need to use StudyTrack

🚀 Getting Started

1

Your Guest ID is auto-generated on first visit. Click the badge in the top-right to view or change it. Save it to restore your data later.

2

Go to My Files and drag-drop or click to upload up to 5 PDFs. Files are tracked by their exact filename.

3

Click Open on any file to jump straight to the PDF Viewer with your notes panel on the right.

🤖 Gemini API Setup

1

Visit aistudio.google.com → API Keys and click Create API Key. It's free.

2

Open app.js in any text editor. Near the very top, find:
const GEMINI_API_KEY = "YOUR_KEY_HERE";

3

Replace YOUR_KEY_HERE with your actual key. Save the file, reload the site. AI features are now active.

4

Model: gemini-2.5-flash — Google's fastest multimodal model that reads entire PDFs in one request.

📋 Managing Topics

1

In Topics, select a file from the sidebar. Click 🤖 AI Extract Topics to let Gemini analyse and classify topics automatically.

2

Check ✓ next to any topic to mark it covered. It moves to the Covered tab. Un-check to move it back.

3

Click ⭐ to mark a topic as important. Use the filter chip to show only important topics.

4

Click 🗑️ to delete any topic. Add new ones manually with the ➕ Add Topic button.

📄 PDF Viewer Tips

1

Use ‹ Prev and Next › to navigate between pages. The current page and total are shown in the toolbar.

2

Use − / + to zoom in or out. The zoom % is shown between the buttons.

3

Click ✨ AI Summary to generate a structured study summary of the open PDF. Requires a Gemini API key.

4

Notes are auto-saved as you type and are linked per file. Switching files swaps the notes panel automatically.

💾 Data & Guest ID

1

PDFs are stored in your browser's IndexedDB (much larger than localStorage). Topics and notes use localStorage.

2

Data is stored locally on your device — it's private and requires no account. Guest limit is 5 files.

3

Uploading a file with the same name as an existing one will reuse that file's notes and topics — no duplicate entries.

🚀 Deploying to GitHub Pages

1

Create a GitHub repo. Upload index.html, styles.css, and app.js to the root.

2

In the repo, go to Settings → Pages → Source, select main branch / root, and save.

3

Your site is live at https://username.github.io/repo-name. No backend server needed — Gemini API is called directly from your browser.
